# Data & AI for Designers and Journalists

An open textbook on reading and building with data, machine learning and AI, treating data two ways at once: as a **material** a designer shapes, and as **evidence** a journalist tests. It accompanies the Data & AI track of the [Master Data-driven Design](https://www.hu.nl/voltijd-opleidingen/master-data-driven-design) and the minor [Big Data and Design](https://husite.nl/minors/minors/big-data-and-design/) at HU University of Applied Sciences Utrecht, but it is organised by concept rather than by week and stands on its own.

**Read it online:** <https://hekman.github.io/data-ai-book/>

No prior programming is assumed. Every chapter pairs plain-language ideas with runnable Python (pandas and friends) and ends with exercises. History and ethics run *beside* the practice through four kinds of sidebar — **Origins**, **In Practice**, **Critical Lens** and **Going Further** — rather than being confined to separate chapters.

## Contents

The book is planned in three parts:

- **Part I · Data as Material** — reading, cleaning, exploring, visualising and collecting real, messy data. *(Available now.)*
- **Part II · Reading the Systems That Learn** — machine-learning literacy: models, evaluation, bias. *(In progress.)*
- **Part III · Building With AI** — designing and shipping interactive, AI-driven tools. *(In progress.)*

Part I currently covers: data structures and variables; the origins of a dataset; the dataframe; cleaning and messy data; combining datasets; exploratory analysis; visualisation as argument; dashboards; and collecting your own data, closing with a project milestone.

## Contributing

Corrections, clarifications and suggestions are welcome. The quickest routes are to open an [issue](https://github.com/Hekman/data-ai-book/issues), or to use the pencil icon on any page of the live site, which links straight to that page's Markdown source for an edit and pull request. Because this is a teaching text, contributions that improve clarity, fix errors in the code examples, or strengthen the historical and ethical sidebars are especially valued.
